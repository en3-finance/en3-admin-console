# Demo Script

Use this script for a short walkthrough of the public reference console.

## 1. Open Dashboard

State that the repo is a public, mock-only reference for institutional
Wallet-as-a-Service operations. Point out organizations, wallets, pending
approvals, risk reviews, and the module coverage list.

## 2. Review Organizations and RBAC

Open Organizations, Users, and Roles / RBAC. Explain that the console shows how
bank, fintech, payment provider, operations, compliance, treasury, and audit
personas could be represented without real auth.

## 3. Review Wallet Registry and Treasury

Open Wallet Registry and Treasury / System Wallets. Explain user, settlement,
treasury, and system wallet categories. State clearly that sweeping,
reconciliation, treasury execution, signing, custody, and ledger infrastructure
are not in this public repo.

## 4. Inspect Policy and Approval

Open Policies and select the high-value stablecoin approval policy. Then open
Pending Approvals and show the transaction detail. Use Approve mock decision or
Reject mock decision to demonstrate session-local decision state.

## 5. Show Simulation and Risk Review

Open Transaction Simulation and run the mock simulation. Then open Risk Review.
State that the signals are mock compliance-readiness examples and no sanctions,
KYT, address-risk, custody, banking, or payment vendor is integrated.

## 6. Show Audit Log and Webhooks

Open Audit Log to show the event timeline. Open Webhooks to show event
subscription examples. State that webhook targets are mock/redacted and no
delivery occurs.

## Closing

Position the repo as a credible public explanation of the En3 control-plane
model, not as production infrastructure.
