# Control Plane Overview

The En3 Wallet Control Plane reference console demonstrates how operations teams could review organizations, users, roles, wallets, policies, approvals, risk reviews, audit logs, and webhooks.

## Modules

| Module | Reference responsibility |
| --- | --- |
| Organizations | Tenant and product configuration in mock form. |
| Users | Admin and operator identities. |
| IAM/RBAC | Roles and permission examples. |
| Wallet Registry | User, treasury, and system wallets. |
| Treasury/System Wallets | Mock operational wallets only. |
| Policies | Approval and review rules in non-enforcing examples. |
| Pending Approvals | Transactions waiting for admin decision. |
| Transaction Simulation | Sandbox pre-execution review output. |
| Risk Review | Mock address-risk and sanctions-review state. |
| Audit Log | Lifecycle events for operations review. |
| Webhooks | Endpoint and event subscription overview. |
