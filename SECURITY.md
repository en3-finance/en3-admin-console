# Security Policy

Do not report vulnerabilities through public issues. Do not disclose
security-sensitive findings publicly.

Production custody, cryptography, signing orchestration, policy enforcement,
risk scoring, ledger infrastructure, treasury execution, reconciliation,
sweeping, webhook delivery, and customer deployments are not contained in En3
public repositories.

Public repositories contain documentation, mock contracts, sandbox examples, and
reference interfaces only. They must not include private keys, seed phrases,
access tokens, production API hosts, customer data, internal deployment
configuration, private RPC URLs, or real infrastructure details.

This repository uses local mock data only. Approval, policy, transaction
simulation, risk review, audit, treasury, and webhook behavior is reference UI
behavior and must not be treated as production security control behavior.

Contact the En3 team through existing partner channels while a dedicated
security contact is being finalized.
